## 2. 分阶段学习计划

### 阶段 1 — 数据入口：理解原始数据 → epoch 的全流程

**目标**：能看懂 `r01` 在做什么，并能在新实验中修改 trigger 码和 epoch 窗口。

| 项 | 内容 |
|---|---|
| **核心概念** | EyeLink ASC 格式（SAMPLE 行 vs MSG 行）、trigger 码解析、双眼平均、cue 锁时切 epoch、校准归一化（为什么用校准而非屏幕物理尺寸）、NaN 扩展处理眨眼 |
| **必读脚本** | `r00_setup.m`（配置表）→ `helper_parse_asc.m`（逐行）→ `r01_prepare_eye_data.m`（逐行） |
| **学习深度** | `helper_parse_asc.m` 和 `r01` 必须逐行读。重点：(1) trigger 码如何映射到 trial 类型；(2) `helper_parse_asc` 如何用 `sscanf` 热路径读取常规样本行、再用 token 慢路径兼容 `.` 缺失值和异常列数；(3) 校准 7 点如何变成 ±100% 归一化；(4) `eye_data` 结构的每个字段是什么 |
| **实践任务** | ① 在 MATLAB 中 `load('normalized_eye_data/s01.mat')`，查看 `eye_data` 结构，画出 trial 1 的 X/Y 时间序列；② 在 `r00_setup.m` 中找到 `cfg.trig` 那行，对照 `.asc` 文件验证 21=左/23=右；③ 把 `cfg.epoch_pre/post` 改成 `[-2, 1]`，重跑 r01，观察 `eye_data.time` 变化；④ 分别设置 `cfg.use_fieldtrip = true` 和 `false`，重跑 r01，对比两种模式的输出 .mat 文件是否一致（校准/blink/平均逻辑相同，应完全一致） |
| **预期产物** | 能解释 `eye_data.trial` 的维度含义、`trialinfo` 的取值集合、`time` 向量的零点对齐 |
| **自检** | ① `eye_data.trialinfo` 取值 = {21,22,23,24}；② 每被试 1200 trials；③ `eye_data.time(1)` = -1（秒） |
| **常见坑** | sess2 文件名有 `b` 后缀；校准 Y 轴点数与 X 轴不同；ASC 缺失值 `.` 不能直接 `sscanf` 成完整数值行，必须回退到 token 解析并转 NaN |

### 阶段 2 — 核心算法：微眼跳检测（最重要）

**目标**：能读懂 `PBlab_gazepos2shift_1D` 的每一行，理解每个参数的物理意义，能在新实验中调参。

| 项 | 内容 |
|---|---|
| **核心概念** | 速度 = 位置差分；中位数 × 阈值法（为何用 median 而非 mean）；高斯平滑稳定速度估计；最小 ISI 防重复计数；threshold-crossing 前后窗位差 = 眼跳幅度 |
| **必读脚本** | `PBlab_gazepos2shift_1D.m`（**逐行**，79 行，全管线最可移植的资产）→ `gazeShiftRateOverSize.m`（逐行）→ `r02_detect_microsaccades.m` |
| **学习深度** | `PBlab` 必须读到能**闭眼复述每个 cfg 参数的物理意义**。重点：`cfg.threshold`（中位数的倍数，不是绝对速度）、`cfg.smoothWin`（高斯窗宽 ms）、`cfg.minISI`（防什么）、`winbef/winaft`（位移怎么算的） |
| **实践任务** | ① 取 `eye_data.trial` 的第 50 个 trial，画出 X 位置 → 速度 → 平滑速度 → 阈值穿越点，手动验证眼跳被正确检测；② 把 `cfg.threshold` 从 3 改到 6，重跑 r02，看 Fig 1c 红色团块如何变弱；③ 把 `cfg.threshold` 改到 1，看假阳性如何爆发；④ 读 `gazeShiftRateOverSize.m` 第 68-69 行，理解 toward/away 对称化为什么要 `abs` |
| **预期产物** | 能在脑中建立"原始位置 → 速度 → 阈值 → 眼跳事件"的完整数据流图 |
| **自检** | ① Fig 1c 红色团块在 < 20% shift size、300-500 ms；② threshold=6 时团块明显变弱；③ threshold=1 时全图变红（假阳性爆炸） |
| **常见坑** | 把 `cfg.threshold` 理解为绝对速度阈值（实际是 median 的倍数）；混淆 t=0 的含义（cue onset，不是 encoding） |

### 阶段 3 — 条件映射：Trial 分类与行为对齐

**目标**：能理解 toward/away/noMS 的判定逻辑，能在新范式中修改分类窗口和方向映射。

| 项 | 内容 |
|---|---|
| **核心概念** | "首事件方向"判定（取分类窗口内的第一个眼跳）；过小眼跳的特殊处理（< 1° 不算 toward/away）；不可用 trial（epoch 内有 NaN）；行为 log 与 eye trial 的逐行对齐；RT 修剪（> 3000 ms + 2.5 SD 截断） |
| **必读脚本** | `sortTrial_onSaccade.m`（原作者版，对照）→ `r03_sort_trials.m`（我们的版本）→ `r04_link_behavior.m` |
| **学习深度** | `r03` 中嵌套 if 的边界处理（第 65-90 行）是**最容易在自己数据上踩坑的地方**，必须逐行理解 |
| **实践任务** | ① 加载 `event_TAN_mini1/s01.mat`，验证 toward + away + noMS + tooSmall + unusable = 1200；② 统计 toward 占比，应 ≈ 44%（论文值）；③ 把分类窗口从 [0.2, 0.6] 改到 [0.3, 0.5]，看比例如何变化；④ 加载 `behavior_aligned/s01.mat`，检查 RT 的分布和修剪后 trial 数 |
| **预期产物** | 能解释为什么 toward 占比 44% 而不是 50%（方向概率 + 过小眼跳 + 无眼跳 trial 的存在） |
| **自检** | ① 五个 logical 向量之和 = 总 trial 数；② toward ≈ 44%；③ `behavior` 表行数 = `eye_data.trial` 行数 |
| **常见坑** | 行为 log 与 eye trial 数不一致（epoch 越界丢 trial）；首眼跳过小但后续有大眼跳 → 按论文逻辑算 tooSmall |

### 阶段 4 — 群体统计：从描述到推断

**目标**：能读懂 cluster permutation 和 RM-ANOVA 代码，能在 N≥5 时运行它们出显著性结果。

| 项 | 内容 |
|---|---|
| **核心概念** | 多重比较问题（为什么不能逐时间点做 t 检验）；Cluster-based permutation 五步流程（配对 t → 阈值上聚簇 → t-mass → 翻转符号 N 次 → p = 比观测大的比例）；RM-ANOVA + Bonferroni 事后比较 + Cohen's d (paired)；FieldTrip `ft_timelockstatistics` 的设计矩阵构建（ivar/uvar）和结果解读（stat.mask、stat.posclusters） |
| **必读脚本** | `helper_cluster_perm_1d.m`（逐行）→ `r06_group_rate_clusterperm.m` → `r05_behavior_analysis.m`（特别是 `run_one_way_rmANOVA` 子函数） |
| **学习深度** | Cluster permutation 必须能**口头复述五步**。这是泛用工具——回到自己数据 100 行内可复用。FieldTrip 模式下需要理解 `cfg.design` 的配对设计矩阵（ivar=2 表示条件，uvar=1 表示被试）。RM-ANOVA 会用即可，不需要理解底层矩阵运算 |
| **实践任务** | ① 把 `cfg.subj` 加到 5 个被试，重跑 r02 + r06，观察 cluster 是否出现（N=5 时可能边缘显著）；② 把 `nPerm` 从 5000 改到 100，看 p 值稳定性变差；③ 阅读 `r05` 的输出，理解 `stats.err.F / p / posthoc` 的含义；④ 分别设置 `cfg.use_fieldtrip = true` 和 `false`，重跑 r06，对比两种模式的显著性结果（随机性可能导致微小差异） |
| **预期产物** | 能解释 Fig 1b 上那条黑横线是什么（cluster 的时间范围）；能解释 Fig 2 柱状图上星号的含义 |
| **自检** | ① 能用自己的话解释 cluster permutation 为什么能控制 type-I error；② 能解释为什么 N<3 时代码跳过统计（安全网机制）；③ 能解释 N=23 与 N=5 时统计结果的差异 |
| **常见坑** | 翻转符号时对原数据翻转（应该对配对差翻转）；cluster 阈值用双侧临界 t 但 t-mass 保留符号 |

## 5. 风险与依赖

| 风险 | 表现 | 解决策略 |
|---|---|---|
| **自己实验的 trigger 码不同** | trial 数对不上、分类全错 | 改 `cfg.trig`，先用 1 个 trial 全程绘图核对 |
| **没有校准模块** | 无法做 ±100% 归一化 | 用屏幕物理尺寸 + 视距手动换算 px→deg |
| **cluster permutation 写错符号翻转** | p 值永远 ~0.5 | 用已有的 `helper_cluster_perm_1d.m`，不要自己重写 |
| **FieldTrip 路径没加** | `ft_*` 函数找不到 | 确保 `cfg.use_fieldtrip = true` 时 `cfg.add_fieldtrip()` 被调用；或手动 `addpath('C:\MATLABEMM\MATLABAddOnEMM\fieldtrip-20241219'); ft_defaults` |
| **ft_preprocessing 无法读取 .asc** | FieldTrip 报错或返回空数据 | r01 采用混合方案：helper_parse_asc 读取 + ft_redefinetrial 做 epoching |
| **ColorBrewer 路径没加** | `brewermap` 找不到 | `addpath` 到 ColorBrewer 目录 |
| **行为 log 与 eye trial 数不一致** | 索引越界 | `r04` 已有 fallback 截断 + warning；自己数据写更严的 assertion |

## 6. 学习节奏建议

按你的水平（改脚本为主），建议每个阶段：

1. **先读**（逐行读脚本，不懂的行标 `?`）
2. **再改**（完成该阶段的实践任务，都是"改参数看效果"）
3. **再写**（每个阶段末尾尝试自己写一小段验证代码，如 `assert` 语句或简单绘图）
4. **遇到不懂的函数** → `help 函数名` 或 `doc 函数名` 在 MATLAB 里查，这是你最需要练的肌肉记忆

阶段 1-3 是你**复用到自己数据**时一定会改的部分，优先级最高。
阶段 4-5 是你**写论文时**需要的部分，可以后补。
