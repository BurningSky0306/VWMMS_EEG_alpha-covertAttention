## 2. 文件命名规则

### 被试编号

- 格式：`sNN`，NN 为两位数字（01–25）
- 跳过：**s05、s07** 不存在于本地数据中（论文剔除标准）
- 实际被试数：**23 人**（论文 N=23，剔除 2 人后）

### 原始数据文件

| 文件模式 | 含义 | 示例 |
|---|---|---|
| `sNN_vm.asc` | 被试 sNN、session 1 的 EyeLink 眼动原始数据 | `s01_vm.asc` — 被试 01、session 1 |
| `sNNb_vm.asc` | 被试 sNN、**session 2** 的眼动数据（`b` 后缀 = session 2） | `s01b_vm.asc` — 被试 01、session 2 |
| `sNN_vm_DDMMYYYY.txt` | 被试 sNN、session 1 的行为 log，末尾为实验日期 | `s01_vm_04042017.txt` — 2017 年 4 月 4 日采集 |
| `sNNb_vm_DDMMYYYY.txt` | 被试 sNN、session 2 的行为 log（`b` 后缀 = session 2） | `s01b_vm_04042017.txt` |
| `_vm` | 实验代号缩写（Visual-Motor working memory task） | — |

### 中间产物与输出文件

| 文件模式 | 含义 | 生成脚本 |
|---|---|---|
| `normalized_eye_data/sNN.mat` | 被试 sNN 的归一化 eye_data 结构（两 session 合并） | r01 |
| `gaze_shift/sNN.mat` | 被试 sNN 的眼跳事件矩阵 data_shift | r02 |
| `event_TAN_mini1/sNN.mat` | 被试 sNN 的 trial 分类 event 结构 | r03 |
| `behavior_aligned/sNN.mat` | 被试 sNN 的对齐行为表 behavior | r04 |
| `GA_*.mat` | **Group Average** — 跨被试聚合的组级统计结果 | r02/r03/r05/r06 |

### TAN 和 mini1 的含义

- **TAN** = **T**oward / **A**way / **N**o-microsaccade，三种 trial 分类条件的首字母缩写
- **mini1** = **mini**saccade 检测，第 **1** 种参数方案（即论文主方法：PBlab 速度阈值法，threshold=3）
- 该目录名沿用原作者代码中的命名（`sortTrial_onSaccade.m` 中 `outfilename_event = 'event_TAN_mini1'`），表示"用 mini1 参数方案得到的 TAN 事件索引"

### 3.2 code/reproduce/（复现管线脚本）

按执行顺序说明：

---

#### r00_setup.m — 全局配置

- **输入**：无（硬编码仓库路径）
- **做了什么**：定义所有路径（`repo_root`、`data_eye`、`data_log`、各 `out_*` 目录）、被试列表（`cfg.subj`）、采样率、epoch 窗口、trigger 编码、微眼跳检测参数、trial 分类参数；**`cfg.use_fieldtrip` 开关**（true = 使用 FieldTrip, false = 使用自定义实现）；自动创建输出目录；添加 code/ 到 MATLAB path
- **输出**：返回 `cfg` 结构体，被所有后续脚本调用

---

#### r01_prepare_eye_data.m — 预处理：.asc → 归一化 eye_data

- **输入**：`data/exp1_sessN_eye/sNN[b]_vm.asc`（原始 EyeLink 文件）+ `r00_setup` 的 cfg
- **做了什么**（根据 `cfg.use_fieldtrip` 分支）：
  1. 调用 `helper_parse_asc` 解析 .asc → 双眼 X/Y + trigger
  2. 校准归一化：用 7 个校准点（trig 201/203–207/209）的 X 排序自动识别"左 3 / 中 1 / 右 3"，计算 center 和 halfRange，将像素坐标转换为 ±100%（对应 ±5.7°）
  3. **`cfg.use_fieldtrip = true` 时**：用 `ft_preprocessing` 读取 + epoch 原始数据，转换 FieldTrip 格式后做双眼平均、归一化、blink padding（ft_preprocessing 失败时自动回退到自定义路径）
  4. **`cfg.use_fieldtrip = false` 时**：双眼平均 → NaN 扩展 → 手动 epoch（与修改前完全一致）
  5. 拼接 session 1 + session 2
- **输出**：`normalized_eye_data/sNN.mat`（含 `eye_data` 结构：`.trial [ntrl×2×ntime]`、`.trialinfo`、`.time`）

---

#### r02_detect_microsaccades.m — 微眼跳检测 + Fig 1c

- **输入**：`normalized_eye_data/sNN.mat`（来自 r01）
- **做了什么**：
  1. 对每个被试，取 X 通道调用 `PBlab_gazepos2shift_1D`（threshold=3×median）检测眼跳事件
  2. 对每个被试，调用 `gazeShiftRateOverSize` 计算 rate × size × time 矩阵
  3. 跨被试聚合为 `GA_struct`（toward/away/diff）
  4. 绘制 Fig 1c 三联热力图（Toward / Away / Difference）
- **输出**：`gaze_shift/sNN.mat`（`data_shift` 结构）、`results/GA_shift_rateAndsize.mat`、`results/figures/fig1c_rate_size_time.png`

---

#### r03_sort_trials.m — Trial 分类 + Fig 1d

- **输入**：`normalized_eye_data/sNN.mat`（来自 r01）+ `gaze_shift/sNN.mat`（来自 r02）
- **做了什么**：
  1. 在 200–600 ms post-cue 窗口内，取每个 trial 的**第一个**可分辨眼跳（幅度 ≥ 1%）
  2. 根据眼跳方向与 cue 方向的关系，分为 toward / away / noMS / tooSmallMS / unusable 五类
  3. 计算每个被试的各条件比例
  4. 绘制 Fig 1d 堆叠条形图
- **输出**：`event_TAN_mini1/sNN.mat`（`event` 结构含 5 个 logical 向量）、`results/GA_trial_proportions.mat`、`results/figures/fig1d_trial_proportions.png`

---

#### r04_link_behavior.m — 行为对齐

- **输入**：`data/exp1_sessN_log/sNN[b]_vm_*.txt`（行为 log）+ `normalized_eye_data/sNN.mat`（来自 r01，用于核对 trial 数）
- **做了什么**：
  1. 读取两 session 的行为 log（Presentation 输出的 .txt），拼接
  2. 与 eye trial 逐行对齐（若行数不一致则按较小者截断并打印警告）
  3. 提取关键列：targetLoc、RT1（cue→response onset）、reportVsTarget
  4. 计算 error = |reportVsTarget|
  5. RT 修剪：>3000 ms 剔除 + 2.5 SD 截断
- **输出**：`behavior_aligned/sNN.mat`（`behavior` 表，含 RT、error、valid_RT 等列）

---

#### r05_behavior_analysis.m — 行为统计 + Fig 2

- **输入**：`behavior_aligned/sNN.mat`（来自 r04）+ `event_TAN_mini1/sNN.mat`（来自 r03）
- **做了什么**：
  1. 对每个被试，按 toward/no/away 三条件计算 mean error 和 mean RT
  2. 归一化：(val − mean_subj) / mean_subj × 100%
  3. One-way Repeated-Measures ANOVA（N≥3 时）+ Bonferroni 事后配对 t 检验 + Cohen's d
  4. N<3 时自动降级为纯描述统计
  5. 绘制 Fig 2（上排：原始均值±SEM + 被试连线；下排：归一化散点）
- **输出**：`results/GA_behavior_stats.mat`（含 err/rt/stats 结构）、`results/figures/fig2_behavior.png`

---

#### r06_group_rate_clusterperm.m — 跨被试 rate + cluster perm + Fig 1b

- **输入**：`gaze_shift/sNN.mat`（来自 r02）
- **做了什么**：
  1. 对每个被试，将 shift 矩阵二值化 → 50 ms 移动平均 × 1000 → Hz（toward/away 分别计算）
  2. 截取 [-0.2, 1] s 窗口
  3. **`cfg.use_fieldtrip = true` 时**：将 toward/away 数据转为 FieldTrip `timelock` 结构，用 `ft_timelockstatistics` 做配对 cluster permutation（depsamplesT, 5000 次随机化），从 `stat.mask` 提取显著时间点
  4. **`cfg.use_fieldtrip = false` 时**：调用 `helper_cluster_perm_1d` 做配对 cluster-based permutation test
  5. 绘制 Fig 1b：toward/away rate 时间序列 + 95% CI 阴影 + 显著性黑横线
- **输出**：`results/GA_rate_timecourse.mat`（含 GA_to/GA_aw/tvec/out）、`results/figures/fig1b_rate_timecourse.png`

---

#### run_all.m — 管线总入口

- **输入**：可选参数 `'from'` / `'only'`
- **做了什么**：顺序执行 r01 → r02 → r03 → r04 → r05 → r06，支持从某步开始或只跑指定步骤；当 `cfg.use_fieldtrip = true` 时预先加载 FieldTrip 路径
- **输出**：调用各脚本，无独立输出

---

#### helper_parse_asc.m — EyeLink ASC 解析器

- **输入**：.asc 文件路径
- **做了什么**：逐行解析 EyeLink ASC 文本，提取双眼 gaze 样本（LX/LY/RX/RY）、trigger（MSG trig 行）、采样率、屏幕分辨率；自动检测单/双眼记录；将缺失值 `.` 转 NaN
- **输出**：`out` 结构（.Fs、.t、.LX/.LY/.RX/.RY、.trig_code/.trig_time、.display、.binocular）

---

#### helper_cluster_perm_1d.m — 1D Cluster Permutation 检验

- **输入**：A、B 两个 [N_subjects × N_timepoints] 配对矩阵
- **做了什么**：Maris & Oostenveld (2007) 方法——配对 t → 超阈值同号相邻点聚簇 → 计算 t-mass → 随机翻转符号 N 次 → 每簇 p = 比例
- **输出**：`out` 结构（.clusters 含 start/stop/mass/p、.sigMask 显著时间点、.tvals 原始 t 值）

---
